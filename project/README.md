# EV Battery SOH Prediction — 4-Model Comparison

NCSU Neural Networks course project. Compares 4 architectures for predicting battery State-of-Health (SOH%) from raw voltage/current/temperature time series extracted from NASA's Randomized Battery Usage Dataset (28 Li-ion batteries).

## Environment

```bash
conda activate torch_env   # Python 3.10, PyTorch + CUDA 11.8
```

## Quickstart

```bash
cd project/

# Step 1: preprocess raw .mat files → HDF5 sequences (~10–30 min)
python src/preprocessing.py \
    --data_dir "../11. Randomized Battery Usage Data Set" \
    --output data/processed/sequences.h5

# Step 2: verify the dataset
python src/dataset.py --h5_path data/processed/sequences.h5

# Step 3: sanity-check each model (shape checks, no data needed)
python src/models/mlp.py
python src/models/cnn1d.py
python src/models/lstm.py
python src/models/transformer.py

# Step 4: train all 4 models
python src/train.py --model mlp         --epochs 100 --device cuda
python src/train.py --model cnn         --epochs 100 --device cuda
python src/train.py --model lstm        --epochs 100 --device cuda
python src/train.py --model transformer --epochs 100 --device cuda

# Step 5: compare results
python src/evaluate.py --compare_all
```

Results land in `results/figures/` and `results/metrics/`.

## Data Pipeline

**Source:** `../11. Randomized Battery Usage Data Set/` (28 batteries, RW1–RW28)

**Preprocessing (`src/preprocessing.py`):**
- Loads each `.mat` file using the existing `battery_soh` package (handles both MATLAB v5 and v7.3 HDF5 formats)
- Identifies reference discharge cycles (used for SOH labels)
- Extracts `charge (random walk)` and `discharge (random walk)` steps only (rest steps are 2 pts, excluded)
- Drops steps shorter than 10 samples
- Resamples each step to exactly 301 time points via linear interpolation
- Labels each step with the SOH from the next reference discharge
- Writes to HDF5: `X` (N, 301, 3), `y` (N,), `battery_id`, `block_index`

**HDF5 schema:**
| Dataset | dtype | shape | description |
|---|---|---|---|
| X | float32 | (N, 301, 3) | [voltage, current, temperature] |
| y | float32 | (N,) | SOH in [0, 100]% |
| battery_id | bytes | (N,) | e.g. b'RW10' |
| block_index | int32 | (N,) | which RW block within this battery |

## Models

All 4 models share the same interface: **(batch, 301, 3) → (batch, 1)**

| Model | Description | Parameters |
|---|---|---|
| **MLP** | Extracts 20 summary stats in `forward()`, passes through FC layers | ~26K |
| **1D-CNN** | 4 conv blocks (32→64→128→128 filters), global avg pool | ~200K |
| **LSTM** | Bidirectional, 2 layers, hidden=128, takes last hidden state | ~450K |
| **Transformer** | 3-layer encoder, d_model=64, 4 heads, mean pool | ~230K |

---

## Train/Test Split

**Battery-level split — no data leakage between batteries.**

| Set | Batteries | Groups |
|---|---|---|
| **Test** | RW13, RW14, RW25, RW26, RW27, RW28 | skewed_low/room_temp + skewed_high/40C |
| **Train** | Remaining 22 batteries | All other groups |

Test batteries were chosen to test OOD generalization:
- **RW25–28**: 40°C elevated temperature (skewed_high) — temperature the model has not fully seen
- **RW13–14**: Room temperature skewed-low distribution — distribution-shape OOD

---

## Training Configuration

| Setting | Value |
|---|---|
| Loss | MSELoss (y scaled to [0,1]) |
| Optimizer | Adam, lr=1e-3, weight_decay=1e-4 |
| Scheduler | ReduceLROnPlateau (patience=5, factor=0.5) |
| Early stopping | patience=10 on val MAE |
| Batch size | 256 (128 for Transformer) |
| LSTM gradient clipping | max_norm=1.0 |
